Email Alerts (Brevo)

This page explains how automatic email notifications are sent when Overwatch enters a major outage state.

Major outage emails are sent by GitHub Actions, not by browser code.
Brevo Actions automation

How It Works

  • The refresh workflow updates `site/data/status.json` every 30 minutes.
  • A secure Python step checks if severity is `major` and decides if an email should be sent.
  • Cooldown and state tracking prevent duplicate sends on every run.
  • Brevo API keys stay in GitHub Secrets and never appear in public frontend files.

Required GitHub Secrets

  • `BREVO_API_KEY`
  • `ALERT_EMAIL_FROM` (verified Brevo sender address)
  • `ALERT_EMAIL_TO` (comma-separated recipients)

Optional: set `ALERT_EMAIL_SENDER_NAME`, `ALERT_MAJOR_COOLDOWN_MINUTES`, and `ALERT_SITE_URL` in the workflow env block.

Public Signup Widget (Captcha)

  • Create a Brevo subscription form with captcha + double opt-in enabled.
  • Copy the public form URL (Brevo/Sibforms hosted link).
  • Set `site/data/subscription.json` -> `form_url` to that URL.
  • The dashboard embeds that secure form so users can subscribe without exposing API keys.

Brevo Setup Checklist

  1. Create a Brevo account and open SMTP & API settings.
  2. Create a new API key for transactional emails.
  3. Add and verify your sender identity in Brevo (single sender or domain).
  4. Add the required secrets in your GitHub repository settings.
  5. Run `Update Site Data` manually once to validate delivery.

Safety Notes

  • Never commit API keys to the repository.
  • Do not store recipient email addresses in frontend JS files.
  • Use separate API keys for production and testing.
